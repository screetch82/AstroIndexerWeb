# AstroIndexer 1.0 RC1 — announcement video

A [Remotion](https://remotion.dev) project that renders the RC1 announcement
video (16:9 for YouTube and 9:16 for Shorts/Reels), brand-matched to the site.
Everything renders **locally** — the only network step is regenerating the
voiceover (edge-tts).

## What's in the box

| Path | What |
|------|------|
| `src/Announcement.tsx` | The composition — intro, 6 feature beats, outro. Responsive to 16:9 / 9:16 via `useVideoConfig`. |
| `src/theme.tsx` | Brand tokens (mirrored from `css/style.css`), starfield background, animation helpers. |
| `src/Root.tsx` | Registers both compositions (`Announcement` 1920×1080, `AnnouncementVertical` 1080×1920). |
| `src/manifest.json` | Generated by `gen_vo.py`: per-beat narration text, audio file, measured duration, and frame timings. The composition times itself to this. |
| `public/shots/` | The RC1 screenshots. |
| `public/vo/` | Voiceover clips (one MP3 per beat). |
| `public/music/bed.wav` | Original synthesized ambient music bed. |
| `gen_vo.py` | Regenerates the voiceover + `manifest.json`. |
| `gen_music.py` | Regenerates the music bed to match the current total length. |
| `gen_captions.py` | Emits `out/*.srt` captions + `out/*-youtube.md` (description + chapters), timed from the manifest. |

## Brand voice

Narration follows the app's `BRAND_VOICE` rules (`tools/tutorial_capture/glossary.py`):
British English, conversational-but-technical, active voice, correct domain
terms (FITS, FWHM, HFR, plate solving, …), and **no marketing puff**
("powerful", "seamlessly", "intelligent", "robust", …). Delivered at the shipped
`+0%` rate.

## Voice & music provenance

- **Voice:** edge-tts, `en-GB-LibbyNeural` — the same neural voice used by the
  app's tutorial narration pipeline (`tools/tutorial_capture`). Regenerating
  requires network (Microsoft edge endpoint); it's free, no API key.
- **Music:** 100% synthesized by `gen_music.py` (numpy) — an original ambient
  pad progression, so there are **no third-party samples or licensing
  concerns**. Regenerate offline any time.

## Render

```bash
npm install                      # Remotion + React

# Renders straight from the committed assets — no regeneration needed:
npm run render                   # -> out/announcement.mp4  (16:9)
npx remotion render src/index.ts AnnouncementVertical out/short.mp4 \
  --codec=h264 --image-format=png --crf=16   # 9:16

npm run studio                   # live preview / scrub in the browser
```

## Change the script or timing

1. Edit the `SEGMENTS` narration in `gen_vo.py` (and/or `RATE`).
2. `python3 gen_vo.py`   (needs `pip install edge-tts`) → rewrites the VO + `manifest.json`.
3. `python3 gen_music.py` (needs `pip install numpy`) → re-fits the music bed to the new length.
4. `npm run render`.
5. `python3 gen_captions.py` → refresh `out/*.srt` + `out/*-youtube.md` to match.

Feature copy (titles/bodies/chips) lives in `BEATS` in `src/Announcement.tsx`.
