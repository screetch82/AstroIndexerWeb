# AstroIndexer 1.0 RC1 — announcement video

A [Remotion](https://remotion.dev) project that renders the RC1 announcement
video (16:9 for YouTube and 9:16 for Shorts/Reels), brand-matched to the site.
Everything renders **locally** — the only network step is regenerating the
voiceover (edge-tts).

## What's in the box

| Path | What |
|------|------|
| `src/Announcement.tsx` | The composition — intro, 6 feature beats, outro. Responsive to 16:9 / 9:16 via `useVideoConfig`. |
| `src/theme.tsx` | Brand tokens (mirrored from `css/style.css`), starfield background, animation helpers. |
| `src/Root.tsx` | Registers both compositions (`Announcement` 1920×1080, `AnnouncementVertical` 1080×1920). |
| `src/manifest.json` | Generated by `gen_vo.py`: per-beat narration text, audio file, measured duration, and frame timings. The composition times itself to this. |
| `public/shots/` | The RC1 screenshots. |
| `public/vo/` | Voiceover clips (one MP3 per beat). |
| `public/music/bed.wav` | Original synthesized ambient music bed. |
| `gen_vo.py` | Regenerates the voiceover + `manifest.json`. |
| `gen_music.py` | Regenerates the music bed to match the current total length. |
| `gen_captions.py` | Emits `out/*.srt` captions + `out/*-youtube.md` (description + chapters), timed from the manifest. |

## Brand voice

Narration follows the app's `BRAND_VOICE` rules (`tools/tutorial_capture/glossary.py`):
British English, conversational-but-technical, active voice, correct domain
terms (FITS, FWHM, HFR, plate solving, …), and **no marketing puff**
("powerful", "seamlessly", "intelligent", "robust", …). Delivered at the shipped
`+0%` rate.

## Voice & music provenance

- **Voice:** edge-tts, `en-GB-LibbyNeural` — the same neural voice used by the
  app's tutorial narration pipeline (`tools/tutorial_capture`). Regenerating
  requires network (Microsoft edge endpoint); it's free, no API key.
- **Music:** 100% synthesized by `gen_music.py` (numpy) — an original ambient
  pad progression, so there are **no third-party samples or licensing
  concerns**. Regenerate offline any time.

## Render

```bash
npm install                      # Remotion + React

# Renders straight from the committed assets — no regeneration needed:
npm run render                   # -> out/announcement.mp4  (16:9)
npx remotion render src/index.ts AnnouncementVertical out/short.mp4 \
  --codec=h264 --image-format=png --crf=16   # 9:16

npm run studio                   # live preview / scrub in the browser
```

## Change the script or timing

1. Edit the `SEGMENTS` narration in `gen_vo.py` (and/or `RATE`).
2. `python3 gen_vo.py`   (needs `pip install edge-tts`) → rewrites the VO + `manifest.json`.
3. `python3 gen_music.py` (needs `pip install numpy`) → re-fits the music bed to the new length.
4. `npm run render`.
5. `python3 gen_captions.py` → refresh `out/*.srt` + `out/*-youtube.md` to match.

Feature copy (titles/bodies/chips) lives in `BEATS` in `src/Announcement.tsx`.

## Feature tutorials (pilot)

`TutorialBackfocus` is the first feature-tutorial composition and the template for
the rest (Planning, Quality Analytics, Equipment Manager, …). It's a **zoom-to-highlight
walkthrough**: the camera pans/zooms across an annotated screenshot, step by step, with
animated lower-third captions, brand-voice narration, and music.

- `gen_tutorial.py` — narration (edge-tts LibbyNeural) + `src/tutorial_backfocus.json`
  (per-step `region` to zoom to + `caption` + timings). Edit `STEPS` to author a tutorial.
- `src/Tutorial.tsx` — the composition. `target(region)` centres + zooms on a region and
  **clamps the pan so the image always covers the frame** (no black edges). Regions are
  `[x, y, w, h]` fractions of the screenshot.
- `gen_music.py <manifest.json> <out.wav>` — score any manifest (reused for the tutorial bed).
- `whisper_srt.py <media> <out.srt>` — **stage F**: transcribe *any* audio (recorded
  screencasts included) with faster-whisper → SRT. Note: for **scripted** narration the
  manifest-derived SRT is exact; Whisper's value is **unscripted/recorded** footage
  (it trips on domain terms like "spacers"/"AstroIndexer" from mixed audio).

```bash
python3 gen_tutorial.py                                  # VO + manifest
python3 gen_music.py src/tutorial_backfocus.json public/music/tutorial_bed.wav
npx remotion render src/index.ts TutorialBackfocus out/tutorial.mp4 \
  --codec=h264 --image-format=png --crf=16
python3 whisper_srt.py out/tutorial.mp4 out/tutorial.srt  # optional (recorded footage)
```

## Live screencasts (compositing)

For a real walkthrough, the base track is a **live screen recording** (the app's own
`tools/tutorial_capture` pipeline — macOS `avfoundation` recorder is implemented and
verified working here) instead of a screenshot. Remotion then supplies a **transparent
graphics track** composited over that footage with ffmpeg:

- `src/Overlay.tsx` — graphics only (header, animated lower-third, progress), no background.
  Render **with alpha** (PNG frames + a yuva pixel format, since `remotion.config` uses
  jpeg for the opaque comps):
  ```bash
  npx remotion render src/index.ts Overlay out/overlay.mov \
    --codec=prores --prores-profile=4444 --image-format=png --pixel-format=yuva444p10le
  ```
- `compose_screencast.sh BASE.mp4 OVERLAY.mov VO.mp3 MUSIC.wav OUT.mp4` — ffmpeg glue:
  `overlay` the alpha track over the base, `amix` narration + ducked music, export H.264/AAC.

Swap `BASE.mp4` for a live `avfoundation` recording and it's a real screencast. The one
interactive step left is driving the app through a feature journey while recording. See
`PIPELINE_PROPOSAL.md`.
